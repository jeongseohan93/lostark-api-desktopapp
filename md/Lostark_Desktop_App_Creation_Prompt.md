넌 로스트아크를 하는 유저고, 아직 캐릭터는 더 성장해야 해. 레이드 숙제, 재련 비용, 거래소 시세, 경매장 매물, 생활 효율, 주간 골드처럼 매일 확인해야 하는 정보가 흩어져 있어서 성장 루틴이 번거롭다. 그래서 로스트아크 오픈 API를 활용해 내 캐릭터 성장과 골드 운용을 편하게 관리할 수 있는 데스크톱 프로그램을 제작하려고 한다.

# 로스트아크 성장 편의 데스크톱 앱 제작 프롬프트

## 목표

로스트아크 유저가 캐릭터 성장에 필요한 정보를 한 화면에서 빠르게 확인하고, 거래소/경매장/캐릭터/콘텐츠 데이터를 기반으로 효율적인 의사결정을 할 수 있는 Electron 데스크톱 앱을 만들어라.

앱은 단순 조회 도구가 아니라, 성장 중인 유저가 매일 켜두고 사용하는 실전형 보조 프로그램이어야 한다. 핵심 가치는 다음과 같다.

- 로스트아크 Open API 데이터를 안전하게 불러온다.
- 거래소/경매장 가격을 기반으로 비용과 효율을 계산한다.
- 내 캐릭터, 원정대, 레이드 골드, 재련 재료, 즐겨찾기 아이템을 관리한다.
- 반복 확인이 필요한 시세와 콘텐츠 정보를 대시보드에서 요약한다.
- 모든 UI는 한국어 기준이며, 게임 유저가 바로 이해할 수 있는 용어를 사용한다.

## 기술 스택

- Electron 데스크톱 앱
- Vite + React + TypeScript
- React Router 기반 페이지 라우팅
- CSS Modules 기반 스타일링
- Axios 기반 Lost Ark Open API 호출
- Electron IPC로 렌더러와 메인 프로세스 통신
- Prisma + SQLite로 API 키와 로컬 설정 저장
- Recharts로 시세/추이 차트 구현
- react-icons로 사이드 메뉴와 버튼 아이콘 구성

## 기본 앱 구조

앱을 실행하면 API 키가 저장되어 있는지 확인한다.

- API 키가 없으면 API 키 입력 화면을 먼저 보여준다.
- API 키가 있으면 대시보드로 진입한다.
- API 키는 렌더러에 직접 하드코딩하지 말고 Electron 메인 프로세스와 Prisma SQLite를 통해 저장/조회한다.
- 모든 Lost Ark Open API 요청은 `authorization: bearer {JWT}` 헤더를 사용한다.
- GET 요청은 `accept: application/json`, POST 요청은 `content-type: application/json`을 포함한다.
- API 요청 실패, 점검 시간, 요청 제한, 빈 응답을 UI에서 명확히 안내한다.

## 사이드 메뉴

좌측 사이드바는 접기/펼치기를 지원하고, 상위 카테고리 중심으로 구성한다. 모든 기능을 1차 메뉴에 펼쳐두지 말고, 비슷한 기능은 카테고리 하위 메뉴나 화면 내부 탭으로 묶는다.

- `/` 대시보드
- 거래/시세
  - `/trade/market` 거래소
  - `/trade/auction` 경매장
  - `/trade/favorites` 즐겨찾기
  - `/trade/crystal` 크리스탈 시세
  - `/trade/sniper` 경매장 스나이핑
- 캐릭터/성장
  - `/growth/character` 캐릭터 검색
  - `/growth/enhancement` 재련 계산기
  - `/growth/combat-power` 전투력 추정
  - `/growth/weekly-gold` 주간 골드
- 효율/계산
  - `/calc/package` 패키지 효율
  - `/calc/life-energy` 생활의 기운 효율
  - `/calc/abidos-fusion` 아비도스 융화 효율
  - `/calc/fee` 수수료 계산기
  - `/calc/raid-auction` 레이드 경매 배분기
- 설정
  - API 키 삭제/재등록
  - 앱 설정 관리

사이드바에는 `대시보드`, `거래/시세`, `캐릭터/성장`, `효율/계산`, `설정`만 1차 메뉴로 노출한다. 카테고리를 클릭하면 하위 메뉴가 펼쳐지고, 현재 선택된 하위 메뉴는 활성 상태로 표시한다.

비슷한 화면은 다음처럼 UI를 공유한다.

- 거래소, 경매장, 즐겨찾기, 크리스탈 시세, 경매장 스나이핑은 `거래/시세` 카테고리 안에서 검색/시세/알림 중심 레이아웃을 공유한다.
- 캐릭터 검색, 재련 계산기, 전투력 추정, 주간 골드는 `캐릭터/성장` 카테고리 안에서 캐릭터 기준 성장 관리 흐름으로 묶는다.
- 패키지 효율, 생활의 기운 효율, 아비도스 융화 효율, 수수료 계산기, 레이드 경매 배분기는 `효율/계산` 카테고리 안에서 계산 도구형 레이아웃을 공유한다.
- 같은 카테고리 안의 화면 전환은 사이드바 하위 메뉴 또는 콘텐츠 상단 탭으로 처리한다.

## 페이지별 기능 요구사항

### 대시보드

메인 화면은 유저가 앱을 켰을 때 가장 먼저 확인해야 하는 정보를 요약한다.

- 로스트아크 공지사항 목록 표시
- 진행 중인 이벤트 표시
- 오늘의 모험섬 표시
- 필드보스와 카오스게이트 일정 표시
- 즐겨찾기 거래소 아이템과 경매장 옵션 매물의 최신 시세 요약
- 앱 시작 시 갱신된 즐겨찾기 시세 변동 요약
- 거래소 페이지로 빠르게 이동하는 버튼 제공
- API 로딩/에러/빈 데이터 상태 처리

### 거래소

거래소 아이템을 검색하고 가격 판단을 돕는 화면을 만든다.

- `GET /markets/options`로 카테고리와 검색 옵션을 불러온다.
- `POST /markets/items`로 아이템명, 등급, 카테고리, 페이지 조건을 검색한다.
- 아이템명, 등급, 현재 최저가, 최근 거래가, 묶음 수량, 판매 수량 등을 표로 표시한다.
- 즐겨찾기 추가 버튼을 제공한다.
- 즐겨찾기 아이템은 앱 시작 시 자동 조회하고 로컬 DB에 가격 스냅샷을 저장한다.
- 저장된 가격 스냅샷은 주식 차트처럼 시간축 기반 그래프로 표시한다.

### 경매장

장신구, 어빌리티 스톤, 팔찌 등 경매장 매물을 조건 기반으로 검색한다.

- `GET /auctions/options`로 경매장 검색 옵션을 불러온다.
- `POST /auctions/items`로 품질, 등급, 부위, 각인, 특성, 가격 조건을 검색한다.
- 매물명, 품질, 등급, 즉시구매가, 입찰가, 만료 시간, 옵션을 카드 또는 표로 표시한다.
- 조건이 복잡하므로 검색 폼은 단계별로 정리하고, 초기화/검색/정렬 기능을 제공한다.
- 원하는 옵션 조합의 경매장 매물을 즐겨찾기로 저장할 수 있다.
- 경매장 즐겨찾기는 동일 조건으로 다시 검색해 현재 최저가 매물을 갱신한다.

### 캐릭터 검색

캐릭터 이름을 입력하면 원정대 및 장비 정보를 조회한다.

- `GET /characters/{characterName}/siblings`로 원정대 캐릭터 목록을 표시한다.
- `GET /armories/characters/{characterName}/profiles`로 서버, 직업, 아이템 레벨, 원정대 레벨, 칭호 등을 표시한다.
- 장비, 각인, 보석, 전투 특성, 기본 특성, 성향 정보를 탭 또는 섹션으로 정리한다.
- 캐릭터 이름은 URL 인코딩 처리한다.
- 존재하지 않는 캐릭터와 API 실패를 구분해 안내한다.

### 패키지 효율

상점 패키지나 이벤트 패키지의 골드 환산 효율을 계산한다.

- 패키지 가격과 구성품을 입력한다.
- 구성품 이름으로 거래소 최저가를 조회한다.
- 구성품별 수량, 단가, 총가치를 계산한다.
- 패키지 가격 대비 이득/손해, 골드 환산 효율, 추천 여부를 표시한다.
- 직접 가격 입력도 지원해 API 검색 실패 상황에 대응한다.

### 생활의 기운 효율

생활 콘텐츠별 골드 효율을 비교한다.

- 활동명, 사용한 생활의 기운, 획득 아이템과 수량을 입력한다.
- 획득 아이템은 거래소 최저가 기준으로 환산한다.
- 총 수익, 생활의 기운 1당 골드, 1,000 생기당 골드를 계산한다.
- 여러 활동 기록을 저장하고 가장 효율이 높은 활동을 강조한다.

### 아비도스 융화 재료 효율

아비도스 융화 재료를 거래소에서 바로 구매하는 경우와 직접 생활을 통해 재료를 수급해 제작하는 경우의 효율을 비교한다.

- 계산 대상 아비도스 융화 재료 종류와 필요 수량을 선택한다.
- 거래소 구매 방식은 `POST /markets/items`로 아비도스 융화 재료의 현재 최저가를 조회해 총 구매 비용을 계산한다.
- 직접 생활 방식은 고고학, 낚시, 수렵 등 생활 활동별 재료 수급량을 입력하고, 재료의 거래소 시세 또는 직접 획득 기준 원가를 반영한다.
- 제작에 필요한 생활 재료, 제작 수수료, 영지 제작 비용, 대성공 기대값을 입력하거나 기본값으로 관리한다.
- 거래소 구매 비용, 직접 생활 제작 비용, 절감 골드, 생활의 기운 1당 기대 골드, 추천 방식을 비교한다.
- 직접 생활한 재료도 거래소에 팔 수 있다는 기회비용을 반영해 실제 이득과 체감 이득을 분리해서 보여준다.

### 전투력 추정

캐릭터 성장 데이터를 기반으로 전투력 추이를 추정한다.

- 아이템 레벨과 전투력 데이터를 입력/저장한다.
- 수집된 데이터를 표와 차트로 표시한다.
- 회귀 분석이나 추세선을 통해 목표 레벨의 예상 전투력을 계산한다.
- 데이터가 부족할 때는 분석 신뢰도를 낮게 안내한다.

### 즐겨찾기

자주 보는 거래소 아이템과 원하는 옵션의 경매장 매물을 모아 관리한다.

- 거래소 검색 결과에서 별 버튼으로 아이템을 추가한다.
- 경매장 검색 결과 또는 조건 폼에서 원하는 옵션 조합을 즐겨찾기로 추가한다.
- 앱이 켜지면 즐겨찾기 대상들을 자동으로 조회하고 로컬 DB에 가격 스냅샷을 저장한다.
- 거래소 즐겨찾기는 아이템 최저가, 최근 거래가, 묶음 단가를 갱신한다.
- 경매장 즐겨찾기는 저장된 옵션 조건으로 현재 최저가 매물을 다시 검색하고 대표 매물을 갱신한다.
- 즐겨찾기 목록에서 현재가, 이전가 대비 증감, 마지막 갱신 시간, 최저가 매물 정보를 표시한다.
- 가격 히스토리는 로컬 DB에 누적 저장하고, 주식 그래프처럼 시간축 차트로 변화 추이를 보여준다.
- 차트는 기간 필터, 툴팁, 최고가/최저가, 기준가 라인을 제공한다.
- 필요 없는 아이템은 삭제할 수 있다.

### 주간 골드

원정대의 주간 레이드 골드 수익을 계산한다.

- 캐릭터명과 아이템 레벨을 입력해 캐릭터 목록을 만든다.
- 캐릭터별 입장 가능한 레이드를 자동 판단한다.
- 레이드별 관문 골드를 체크박스로 선택/해제한다.
- 원정대 기준 상위 6캐릭터 골드 제한을 반영한다.
- 레이드 골드는 수동 수정 가능해야 한다.
- 총 주간 예상 골드와 캐릭터별 예상 골드를 표시한다.

### 재련 계산기

강화에 필요한 기대 비용을 계산한다.

- 목표 장비 구간과 현재 단계, 목표 단계를 입력한다.
- 파괴석/수호석/돌파석/융화 재료/명예의 파편/골드 등 재련 재료를 반영한다.
- 거래소 최저가로 재료 단가를 가져오되, 직접 입력도 허용한다.
- 성공 확률, 장인의 기운, 기대 시도 횟수, 평균 비용을 계산한다.
- 재료별 비용과 총 예상 비용을 분리해서 보여준다.

### 경매장 스나이핑

원하는 경매장 매물 조건의 가격 변동을 감시하고, 가격이 상승했을 때 알림으로 알려준다.

- 스나이핑 조건 이름을 등록한다.
- 등급, 카테고리, 품질, 각인 조건, 특성 조건, 기준 가격, 상승 알림 기준을 설정한다.
- 상승 알림 기준은 이전 최저가 대비 상승, 기준가 이상 상승, 퍼센트 상승 중 선택할 수 있다.
- 일정 주기로 `POST /auctions/items`를 호출해 조건에 맞는 매물의 현재 최저가를 확인한다.
- 현재 최저가가 사용자가 정한 기준보다 올랐을 때 데스크톱 알림과 앱 내부 알림으로 알려준다.
- 알림에는 조건명, 이전 최저가, 현재 최저가, 상승폭, 발견 시간을 포함한다.
- 알림 내역과 가격 스냅샷은 로컬 DB에 저장한다.
- API 요청 제한을 고려해 폴링 주기를 무리하게 짧게 잡지 않는다.

### 수수료 계산기

거래소와 경매장 판매 수수료를 계산한다.

- 판매 희망가를 입력하면 수수료와 실수령 골드를 계산한다.
- 목표 실수령 골드 기준 필요한 판매가를 역산한다.
- 여러 판매가를 비교할 수 있는 표를 제공한다.
- 거래소/경매장 수수료 정책이 바뀔 수 있으므로 계산식은 상수로 분리한다.

### 레이드 경매 배분기

레이드 경매에서 입찰가와 파티원 수에 따른 손익을 계산한다.

- 파티원 수, 입찰가, 아이템 예상 가치, 참여자 조건을 입력한다.
- 낙찰자 기준 실제 부담액과 나머지 인원의 분배금을 계산한다.
- 손익분기 입찰가와 추천 최대 입찰가를 표시한다.
- 계산 결과는 즉시 갱신되어야 한다.

### 크리스탈 시세

블루 크리스탈 시세를 추적하고 골드/크리스탈 환산을 돕는다.

- 블루 크리스탈 100개 기준 골드 시세를 조회하거나 수동 입력한다.
- 일정 주기로 가격을 저장하고 최근 히스토리를 차트로 표시한다.
- 현재가, 이전가 대비 증감, 마지막 갱신 시간을 표시한다.
- 골드 -> 크리스탈, 크리스탈 -> 골드 변환기를 제공한다.

## UX/UI 요구사항

- 전체 앱은 최소 1200x700 크기를 기준으로 설계한다.
- 커스텀 타이틀바와 창 최소화/최대화/닫기 버튼을 제공한다.
- 좌측 사이드바와 우측 콘텐츠 영역으로 구성하되, 사이드바는 상위 카테고리와 접히는 하위 메뉴 구조를 사용한다.
- 같은 카테고리 안의 유사 기능은 동일한 검색 영역, 결과 표, 차트, 계산 패널 컴포넌트를 재사용한다.
- 화면 수가 많아 보이지 않도록 비슷한 도구는 콘텐츠 상단 탭이나 세그먼트 컨트롤로 묶는다.
- 현재 선택된 카테고리와 하위 메뉴를 명확히 표시하고, 접힌 사이드바 상태에서도 아이콘과 툴팁으로 위치를 알 수 있게 한다.
- 운영 도구처럼 빠르게 스캔 가능한 밀도 있는 레이아웃을 사용한다.
- 중요한 수치에는 단위 `G`, `%`, `개`, `회`를 명확히 표시한다.
- 로딩, 에러, 빈 상태, API 키 없음, 검색 결과 없음 상태를 모두 디자인한다.
- 버튼에는 가능한 아이콘을 함께 사용한다.
- 표가 많은 화면은 정렬, 스크롤, 고정 헤더를 고려한다.
- 게임 분위기는 살리되 과한 장식보다 실용성을 우선한다.

## 데이터 저장

- API 키는 Prisma SQLite의 settings 테이블에 저장한다.
- 즐겨찾기, 거래소 시세 히스토리, 경매장 조건별 최저가 히스토리, 주간 골드 캐릭터 목록, 스나이핑 조건, 알림 내역, 계산기 입력값은 SQLite 로컬 DB에 저장한다.
- 앱 시작 시 즐겨찾기 대상의 최신 가격을 조회하고 timestamp가 포함된 스냅샷으로 저장한다.
- 주식 그래프형 차트를 위해 가격 스냅샷은 시간, 가격, 수량, 출처, 조건 정보를 함께 저장한다.
- 민감한 API 키는 렌더러 로그에 출력하지 않는다.
- 저장 데이터는 앱 재실행 후에도 유지되어야 한다.

## API 요청 정책

- 옵션 API인 `/markets/options`, `/auctions/options`는 자주 호출하지 말고 캐싱한다.
- 공지/이벤트/캘린더는 앱 진입 시 가져오고 필요할 때 새로고침한다.
- 앱 시작 시 거래소 즐겨찾기와 경매장 옵션 즐겨찾기를 1회 자동 갱신하고 로컬 DB에 저장한다.
- 즐겨찾기와 크리스탈 시세는 적절한 간격으로 폴링한다.
- 경매장 스나이핑은 가격 상승 알림을 위해 조건별 최저가만 비교하고, 요청 제한을 넘지 않도록 조건 수와 폴링 주기를 제어한다.
- `X-RateLimit-Remaining`이 낮으면 자동 요청을 줄이고 사용자에게 안내한다.

## 구현 순서

1. Electron + Vite + React + TypeScript 프로젝트를 구성한다.
2. API 키 입력/저장/삭제 흐름을 만든다.
3. Electron preload에서 안전한 `window.lostarkAPI` IPC 브리지를 노출한다.
4. Lost Ark API 클라이언트를 메인 프로세스에 구현한다.
5. 대시보드 레이아웃, 타이틀바, 카테고리형 사이드바, 중첩 라우팅을 만든다.
6. News, GameContents, Markets, Auctions, Characters, Armories API 래퍼를 구현한다.
7. 거래소, 경매장, 캐릭터 검색 같은 핵심 조회 화면을 먼저 만든다.
8. 패키지 효율, 생활 효율, 아비도스 융화 효율, 재련 계산기, 수수료 계산기, 레이드 경매 배분기 같은 계산 도구를 붙인다.
9. 즐겨찾기, 로컬 DB 시세 히스토리, 주식 그래프형 차트, 크리스탈 추적, 가격 상승 알림형 경매장 스나이핑처럼 주기 갱신 기능을 구현한다.
10. 로딩/에러/빈 상태와 API 요청 제한 대응을 정리한다.
11. 빌드와 패키징을 확인한다.

## 완료 기준

- API 키를 저장하면 앱이 대시보드로 진입한다.
- 사이드바는 상위 카테고리와 하위 메뉴 구조로 동작하고, 모든 하위 메뉴가 정상 라우팅된다.
- 대시보드에서 공지, 이벤트, 오늘의 콘텐츠 정보를 확인할 수 있다.
- 거래소와 경매장 검색이 실제 API 응답으로 동작한다.
- 캐릭터 검색에서 원정대, 프로필, 장비, 각인, 보석 정보를 확인할 수 있다.
- 각 계산기는 입력 즉시 결과를 산출하고, 필요한 경우 거래소 시세를 반영한다.
- 아비도스 융화 재료 효율 계산기는 거래소 구매와 직접 생활 제작의 비용 차이를 보여준다.
- 즐겨찾기는 앱 시작 시 거래소 아이템과 경매장 옵션 매물을 갱신하고 로컬 DB에 시세 히스토리를 저장한다.
- 즐겨찾기와 크리스탈 시세는 주식 그래프형 차트로 히스토리와 변화량을 보여준다.
- 경매장 스나이핑은 원하는 매물 조건의 최저가가 상승했을 때 데스크톱 알림을 보낸다.
- API 오류, 점검, 요청 제한, 검색 결과 없음이 사용자에게 명확히 표시된다.
- 앱을 재실행해도 API 키와 주요 사용자 데이터가 유지된다.
